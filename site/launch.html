The problem

AI agents are blind.

Your AI coding agent can read your codebase, run your tests, and write new features. But it can't see your screen.

When you find a visual bug, you have to describe it in text: "the button is too small on mobile," "there's a layout shift after the spinner," "the nav highlight is wrong." You lose nuance. You lose context. You lose time.

Copy-pasting screenshots helps, but they're disconnected from your narration. The agent gets a wall of images with no timeline, no structure, no indication of what you were looking at when.

The solution

Give it eyes and ears.

markupr records your screen while you talk. When you stop, it runs a pipeline: transcribes your voice with Whisper, detects key moments, extracts the exact frames that match what you were describing, and stitches everything into structured Markdown.

The output is purpose-built for AI coding agents. Every screenshot placed exactly where it belongs. Every issue clearly documented. Your agent gets the same context you had when you found the bug.

With v2.4.0, your agent doesn't even need you to paste anything. It can trigger recordings itself, mid-conversation.

What's new in v2.4.0

MCP server with six tools.

Add markupr as an MCP server and your AI coding agent can capture screenshots, record your screen with voice, and receive structured reports -- all mid-conversation. Three lines of config.

claude_desktop_config.json 
{
  "mcpServers": {
    "markupr": {
      "command": "npx",
      "args": ["-y", "markupr-mcp"]
    }
  }
}
capture_screenshot Grab the current screen instantly. Your agent sees exactly what you see, right now.
capture_with_voice Record screen + microphone for a set duration. Get a structured report back with transcript and frames.
start_recording / stop_recording Begin an interactive session. Narrate while you work, stop when done. The full pipeline runs and returns your report.
analyze_video Process any .mov or .mp4 into structured Markdown with extracted frames and timestamps.
analyze_screenshot Feed a screenshot to the analysis pipeline. Get structured observations your agent can act on.

Works with Claude Code, Cursor, Windsurf, and any MCP-compatible client. Zero install -- npx handles everything.

Three interfaces, one pipeline

Use it however you work.

Desktop App

Menu bar → ⌘⇧F

Lives in your macOS menu bar. Hit the hotkey, narrate what you see, stop, paste. The full GUI experience with session history, annotation tools, and review screen.

CLI

npx markupr analyze ./recording.mov

Same pipeline, no desktop app required. Process screen recordings from CI/CD, scripts, or automation. Point it at a video file and get structured Markdown.

MCP Server

npx markupr-mcp

Your AI agent triggers captures mid-conversation. No copy-paste, no context switching. The agent asks to see your screen, markupr delivers the report.

Under the hood

The pipeline.

Every interface -- desktop, CLI, MCP -- runs the same pipeline.

  • Whisper transcription Local Whisper runs on-device by default. OpenAI Whisper-1 API available as BYOK option for faster, higher-quality transcription.
  • Key-moment detection Transcript analysis identifies the moments that matter -- when you describe a bug, point at a UI element, or call out a specific behavior.
  • ffmpeg frame extraction Pulls the exact video frames that match your key moments. Not periodic screenshots -- precision-timed frames correlated with what you were saying.
  • Structured Markdown generation Sections, headings, timestamps, and screenshots assembled into a document your AI agent already understands. Feed to Claude Code, Cursor, or paste into GitHub and Slack.
  • Open source, MIT licensed No telemetry. No tracking. No analytics. Read the code, fork it, break it, fix it. All processing happens on your machine.

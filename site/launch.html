The problem

Feedback stays trapped in local files.

markupR captures great structured feedback. But what happens after? You copy the markdown, paste it into a GitHub issue, manually format it, tag teammates. For CI/CD? You're on your own.

Every team has different tools -- GitHub Issues, Linear, Slack. Feedback that lives only in local files doesn't reach the people who need to act on it.

The solution

Push feedback where your team works.

v2.5.0 adds output templates so you control the shape of every report. Watch mode monitors a directory and processes recordings as they appear. GitHub Issues and Linear integrations push feedback directly into your issue tracker.

The new GitHub Action brings markupR into CI/CD -- add visual feedback to every pull request automatically. Record what's wrong, and the structured report lands right on the PR.

860 tests. Same MIT license. Same pipeline, now connected to everything.

What's new in v2.5.0

Five features that connect your feedback loop.

Output templates, watch mode, GitHub Issues, Linear, and a GitHub Action. Record once, push everywhere.

.github/workflows/feedback.yml 
name: Visual Feedback
on: [pull_request]
jobs:
  feedback:
    runs-on: ubuntu-latest
    steps:
      - uses: eddiesanjuan/markupr-action@v1
        with:
          video: ./recordings/review.mov
          template: github-issue
Output Templates Control the shape of every report. Use built-in templates or create your own for GitHub Issues, Linear, Slack, or plain markdown.
Watch Mode Monitor a directory for new recordings. markupR processes them as they appear -- no manual intervention required.
GitHub Issues Integration Push structured feedback directly to GitHub Issues. Screenshots, transcript, and timestamps -- all formatted and ready to assign.
Linear Integration Create Linear issues from feedback sessions. Structured reports land in your sprint backlog with full visual context.
GitHub Action Add visual feedback to every PR. The action runs markupR in CI/CD and posts structured reports as PR comments.

All integrations work with the same pipeline: Record → Transcribe → Structure → Push. 860 tests across the full suite.

Four interfaces, one pipeline

Use it however you work.

Desktop App

Menu bar → ⌘⇧F

Lives in your macOS menu bar. Hit the hotkey, narrate what you see, stop, paste. Full GUI with session history and annotation tools.

CLI

npx markupr analyze ./recording.mov

Same pipeline, no desktop app required. Watch mode monitors directories. Point it at a video or let it process recordings as they appear.

MCP Server

npx markupr-mcp

Your AI agent triggers captures mid-conversation. No copy-paste, no context switching. The agent asks to see your screen, markupR delivers the report.

GitHub Action

eddiesanjuan/markupr-action@v1

Add visual feedback to every PR in CI/CD. The action processes recordings and posts structured reports as PR comments automatically.

Under the hood

The pipeline.

Every interface -- desktop, CLI, MCP, GitHub Action -- runs the same pipeline. Record → Transcribe → Structure → Push.

  • Output templates Customizable templates shape the output for your destination: GitHub Issues, Linear tickets, Slack messages, or plain markdown. Create your own or use built-in presets.
  • Watch mode Point markupR at a directory and it processes recordings as they appear. Pairs with screen recording tools for hands-free feedback capture.
  • GitHub Issues + Linear Push structured feedback directly into your issue tracker. Screenshots, transcript, and timestamps land formatted and ready to assign.
  • GitHub Action for CI/CD Add visual feedback to every pull request. The action runs markupR in your workflow and posts structured reports as PR comments.
  • 860 tests, MIT licensed No telemetry. No tracking. No analytics. Open source with comprehensive test coverage across the full pipeline.

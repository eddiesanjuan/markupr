5 new features

Everything that shipped.

01

Output templates.

Five built-in formats. Same recording, different output for different destinations. Markdown for AI agents. JSON for automation. GitHub Issue format for your tracker. Linear format for your team. Jira format for enterprise.

terminal
$ npx markupr analyze --template github-issue ./recording.mov
✓ Transcribed 2m 34s of audio
✓ Extracted 4 key moments
✓ Output: ./feedback-github-issue.md
.md Markdown
.json JSON
GH GitHub Issue
LIN Linear
JIRA Jira
02

Watch mode.

Point markupR at a directory. Every new recording that appears gets processed automatically. Drop a file, get structured output. No commands to run. No buttons to click.

terminal
$ npx markupr watch ./recordings
Watching ./recordings for new files...
▸ New file: bug-report.mov
  Transcribing...
  Extracting frames...
✓ Output: ./recordings/bug-report.md
Waiting for next file...

Combine with --template to control the output format. Combine with --push to automatically create issues from every recording.

Leave it running. Record whenever you find something. Every recording becomes a structured, actionable document.

03

Push to GitHub Issues.

One command. Your feedback becomes a GitHub issue with a structured title, body, labels, and screenshots -- uploaded and embedded. No copy-paste. No browser tabs.

terminal
$ npx markupr push github --repo acme/app ./feedback.md
✓ Uploaded 4 screenshots
✓ Created issue #247: "Login button hidden on mobile viewport"
  https://github.com/acme/app/issues/247

Uses gh CLI for authentication -- if you can push code, you can push issues. Screenshots are uploaded to GitHub and embedded in the issue body. Labels are applied automatically based on feedback type.

04

Push to Linear.

Same idea, different tracker. Feedback becomes a Linear issue with priority, labels, and structured description. Assigned to the right team automatically.

terminal
$ npx markupr push linear --team ENG ./feedback.md
✓ Created ENG-412: "Layout shift after spinner dismiss"
  Priority: High | Label: Bug

Requires a Linear API key (stored securely in your OS keychain). Priority and labels are inferred from the feedback content -- urgent language gets flagged as high priority.

05

GitHub Action.

Visual feedback in CI/CD. Run markupR on recordings checked into your repo or captured during E2E tests. The structured report posts as a PR comment automatically.

.github/workflows/feedback.yml 
name: Visual Feedback
on: [pull_request]

jobs:
  feedback:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: eddiesanjuan/markupr-action@v1
        with:
          recording: './test-recordings/*.mov'
          template: 'github-issue'

Runs the full markupR pipeline inside GitHub Actions. Processes any recordings captured during your CI run and posts the structured feedback directly as a PR comment.

Pairs with E2E test frameworks that capture screen recordings on failure. Your test fails, markupR processes the recording, and the visual context lands on the PR.
